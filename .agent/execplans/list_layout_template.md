# ExecPlan: 一覧ページ用テンプレート（リード構成の共通化）

## メタ情報
- バージョン: v0.1 (2025-12-12 作成)
- オーナー: Codex エージェント
- 対象リポジトリ: ~/work/crm (apps/web)
- スコープ: 一覧ページの共通レイアウト/ガイドライン/チェックリストを整備し、今後の新規・改修でリード画面構成を必ず再利用できる状態にする

## 背景
- レイアウト統一を完了したが、今後の開発でバラつきが再発するリスクがある。
- 共有できるテンプレートやルールが無いため、属人的な実装になりがち。

## ゴール
1. 再利用可能な一覧レイアウトコンポーネント（リード画面基準）を用意する。
2. 最低限の共通パーツ（ツールバー/サマリ表示など）をまとめる。
3. 利用ルールをドキュメント化し、PR チェックリストで検証可能にする。
4. 代表ページ（1〜2画面）をテンプレート適用し、利用例を示す。

## アプローチ
- コンポーネント: `components/layouts/list-page-layout.tsx` を新設し、ヘッダ・サーチカード・一覧カードのスロットを提供。
- サブパーツ: `ListToolbar`, `ListSummaryText` など軽量な UI ユーティリティを追加。
- ドキュメント: `docs/ui/list-layout.md` に使用方法・ルール・コード例を記載。
- チェックリスト: `docs/pr-checklist.md` または `.github/PULL_REQUEST_TEMPLATE.md` に「一覧ページはテンプレート使用か」を追加。
- 導入例: 影響が少ないページ（例: Leads, Activities）でテンプレート適用デモを行う。

## 受け入れ基準
- 新規コンポーネントとドキュメントが存在し、プロジェクト内で参照可能。
- 少なくとも1〜2画面がテンプレート経由で動作し、既存動作を崩していない。
- Lint が通る。
